# История решений - Soft_IPK

## 2025-01-27 - Инициализация проекта
**Проблема**: Отсутствие структурированного контекста для большого многоуровневого проекта
**Подпроект**: Корневой уровень
**Влияние**: Все подпроекты
**Причина**: Проект содержит множество независимых подпроектов без единой системы управления контекстом
**Решение**: Создание файлов контекста для корневого уровня и всех подпроектов
**Статус**: Завершено

### Детали решения:
- Создан PROJECT_CONTEXT.md с описанием архитектуры
- Создан SOLUTIONS_HISTORY.md для отслеживания решений
- Создан CHAT_HISTORY.md для истории переписки
- Создан SYNC_INSTRUCTIONS.md для синхронизации
- Создан .cursorrules для правил проекта
- Создан CHANGELOG.md для истории изменений
- Созданы файлы контекста для ключевых подпроектов:
  - Chip_and_Dip - парсер с веб-интерфейсом
  - ChipDip - расширенный парсер с Docker
  - CRB - бот для работы с банками
  - WSB - бот для бронирования рабочих мест
  - T-Helper - AI помощник с векторным поиском

### Влияние на подпроекты:
- **Парсинг проекты**: Унификация подходов к парсингу
- **Telegram боты**: Стандартизация структуры ботов
- **Веб-приложения**: Единые подходы к разработке
- **AI системы**: Общие принципы работы с векторными базами
- **Системы управления**: Унификация архитектуры

### Выявленные связи между подпроектами:
- **Группа парсинга**: Chip_and_Dip ↔ ChipDip (разные технологии одного сайта)
- **Группа ботов**: CRB ↔ WSB ↔ NewBot (похожая архитектура)
- **Группа AI**: T-Helper ↔ RAGFlow (векторный поиск)
- **Группа веб-приложений**: Cristal ↔ dash (работа с данными)
- **Версионность**: WSB v.3 ↔ WSB v.4, Zotero ↔ zotero_web_importer_docker

## Планируемые решения:

### 1. Унификация структуры проектов
**Проблема**: Различные подходы к структуре в разных подпроектах
**Планируемое решение**: Создание единых шаблонов для каждого типа проекта

### 2. Устранение дублирования
**Проблема**: Множественные версии одних и тех же проектов (WSB v.3, v.4)
**Планируемое решение**: Анализ и консолидация версий

### 3. Стандартизация зависимостей
**Проблема**: Различные версии зависимостей в разных проектах
**Планируемое решение**: Создание единого requirements.txt для совместимых проектов
