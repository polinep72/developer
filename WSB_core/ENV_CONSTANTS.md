# Единые переменные окружения для бота и портала

Используйте эти имена в `.env` (корень `WSB_core/.env` и при необходимости локальные `.env` в подпроектах).

## База данных
- `DB_HOST` — хост БД (пример: `192.168.1.22`)
- `DB_PORT` — порт БД (по умолчанию `5432`)
- `DB_NAME` — имя БД (пример: `RM`)
- `DB_USER` — пользователь БД
- `DB_PASSWORD` — пароль БД

## Общие настройки уведомлений
- `WSB_TIMEZONE` — таймзона планировщика/уведомлений (пример: `Europe/Moscow`)
- `WSB_NOTIFICATION_BEFORE_START_MINUTES` — за сколько минут до начала слать уведомление (по умолчанию `10`)
- `WSB_NOTIFICATION_BEFORE_END_MINUTES` — за сколько минут до конца слать уведомление (по умолчанию `10`)

## Бот
- `TELEGRAM_BOT_TOKEN` — токен бота
- `WSB_BOT_SKIP_PENDING` — пропускать старые обновления при запуске (true/false, по умолчанию `true`)

## Портал
- `WSB_PORTAL_HOST` — хост, на котором поднимается портал (пример: `192.168.1.139` или `0.0.0.0`)
- `WSB_PORTAL_PORT` — порт запуска портала в тесте/разработке (пример: `8088`)

## Email-уведомления (общие для бота и портала через wsb_core)
- `SMTP_HOST` — SMTP сервер (пример: `smtp.gmail.com`)
- `SMTP_PORT` — порт SMTP сервера (пример: `587`)
- `SMTP_USER` — пользователь SMTP
- `SMTP_PASSWORD` — пароль SMTP
- `SMTP_FROM_EMAIL` — email отправителя (по умолчанию используется `SMTP_USER`)
- `SMTP_FROM_NAME` — имя отправителя (по умолчанию `WSB System`)
- `ENABLE_EMAIL` — включить/выключить отправку email (true/false, по умолчанию `false` для разработки)
