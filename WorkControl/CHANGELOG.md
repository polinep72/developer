# CHANGELOG

Все значимые изменения в этом проекте будут документироваться в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
и этот проект придерживается [Semantic Versioning](https://semver.org/lang/ru/).

## [1.0.0] - 2025-11-26

### Добавлено
- Мониторинг Docker-контейнеров с проверкой статуса и health
- Мониторинг HTTP endpoints с проверкой доступности и статус-кодов
- Автоматические уведомления в Telegram при проблемах и восстановлении
- Ежечасные проверки всех сервисов
- Ежедневные отчеты о состоянии всех сервисов
- Хранение конфигурации сервисов в PostgreSQL
- Telegram-бот с интерактивным управлением
- Команды бота: `/start`, `/status`, `/check`, `/reload`, `/db`, `/restart`, `/help`
- Инлайн-кнопки для быстрого доступа к функциям
- Перезапуск Docker-контейнеров через Telegram-бота
- Автоматический перезапуск контейнеров (опционально)
- История проверок в базе данных
- Мониторинг новых записей в БД

### Изменено
- Конфигурация сервисов перенесена из `.env` в PostgreSQL
- Упрощен `.env` — только параметры подключения к БД и Telegram
- Улучшена обработка ошибок и логирование

### Исправлено
- Предупреждения при сборке Docker-образа
- Проблемы с запуском Telegram-бота
- Ошибки при загрузке сервисов из БД
- Проблемы с инициализацией базы данных

### Технические детали
- Python 3.9+
- Docker + Docker Compose
- PostgreSQL для хранения конфигурации
- python-telegram-bot для интерактивного бота
- Автоматическое создание таблиц БД при первом запуске
- Миграция сервисов из .env в БД при пустой таблице
