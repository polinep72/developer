# SYNC_INSTRUCTIONS.md

## Инструкции по синхронизации

### При работе на новом компьютере

1. **Клонирование репозитория:**
   ```bash
   git clone <repository-url>
   cd WorkControl
   ```

2. **Чтение файлов контекста:**
   - `PROJECT_CONTEXT.md` — описание проекта, структура, статус
   - `SOLUTIONS_HISTORY.md` — история решенных проблем
   - `CHAT_HISTORY.md` — история переписки с AI-ассистентом
   - `CHANGELOG.md` — история изменений проекта
   - `README.md` — документация проекта

3. **Настройка окружения:**
   - Создать файл `.env` с параметрами подключения к БД и Telegram
   - Убедиться, что PostgreSQL доступен
   - Проверить доступ к Docker socket

4. **Запуск:**
   ```bash
   docker compose up -d --build monitor
   docker logs -f my-monitor
   ```

### При синхронизации между подпроектами

Проект WorkControl является одноуровневым, подпроектов нет.

### При обновлении конфигурации

1. **Изменение сервисов в БД:**
   - Подключиться к PostgreSQL
   - Обновить таблицу `monitoring_services`
   - Выполнить `/reload` в Telegram-боте или перезапустить контейнер

2. **Изменение переменных окружения:**
   - Обновить `.env`
   - Перезапустить контейнер: `docker restart my-monitor`

3. **Изменение кода:**
   - Внести изменения в `main.py`
   - Пересобрать контейнер: `docker compose up -d --build monitor`

### Рекомендации

- Всегда читать файлы контекста перед началом работы
- Обновлять `CHANGELOG.md` при внесении изменений
- Обновлять `SOLUTIONS_HISTORY.md` при решении проблем
- Обновлять `CHAT_HISTORY.md` при значимых обсуждениях
- Коммитить изменения в Git после завершения работы
