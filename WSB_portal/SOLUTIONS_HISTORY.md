## 2025-11-26 — Расширение аналитики и улучшение интерфейса статистики
- **Проблема:** В подвкладках "Статистика по оборудованию" и "Статистика по пользователям" отображалась только базовая информация (топ по часам), не хватало детальной аналитики для принятия решений. Блок "Использование по категориям" показывал "Без категории" вместо конкретного оборудования. В "Пиковых инцидентах" отображалось недогруженное оборудование вместо перегруженного. Графики выходили за границы блоков.
- **Подпроект:** WSB_portal
- **Влияние:** Пользователи аналитики, администраторы, принятие решений по расширению парка оборудования
- **Причина:** Необходимость более детальной аналитики для понимания паттернов использования оборудования и работы пользователей.
- **Решение:** 
  - Расширена аналитика для оборудования: добавлены количество бронирований, средняя длительность сеанса, процент загрузки, распределение по дням недели, пиковые часы использования, динамика по месяцам, топ пользователей для каждого оборудования
  - Расширена аналитика для пользователей: добавлены количество бронирований, средняя длительность сеанса, часто используемое оборудование, распределение по дням недели, пиковые часы работы, динамика активности, разнообразие оборудования
  - Исправлен блок "Использование по категориям": теперь показывает конкретное оборудование с часами работы
  - Исправлена логика "Пиковых инцидентов": теперь показываются дни с перегрузкой (>10 часов), а не недогруженное оборудование
  - Добавлен блок "ВЫВОД:" в конце "Пиковых инцидентов" с рекомендациями по расширению парка оборудования
  - Исправлено отображение графиков: добавлены настройки autosize, margin, поворот подписей для корректного отображения в контейнерах
  - Изменено расположение блоков в расширенной статистике: вертикальное расположение вместо сетки
- **Статус:** выполнено.

## 2025-11-26 — Расширение интервала времени бронирования
- **Проблема:** Фактическое время начала работы не соответствовало предлагаемому интервалу. Интервал бронирования был ограничен 8:00-20:00, что не покрывало реальные потребности пользователей.
- **Подпроект:** WSB_portal
- **Влияние:** Все пользователи, планирование работы с оборудованием
- **Причина:** Необходимость расширить доступное время для бронирования оборудования.
- **Решение:** Изменены константы `BOOKING_START_TIME` (7:00) и `BOOKING_END_TIME` (22:00) в `app/services/bookings.py`. Интервал времени бронирования расширен с 8:00-20:00 до 7:00-22:00 с шагом 30 минут. Для сегодняшней даты показываются только будущие временные слоты, для будущих дат - все слоты с 7:00.
- **Статус:** выполнено.

## 2025-11-26 — Восстановление расширенной аналитики и исправление SQL запросов
- **Проблема:** Расширенная аналитика с подвкладками была потеряна при откате изменений. SQL запросы в dashboard.py использовали несуществующие таблицы и поля, что приводило к ошибкам и использованию демо-данных вместо реальных.
- **Подпроект:** WSB_portal
- **Влияние:** Пользователи аналитики, корректность отображения данных
- **Причина:** Откат изменений удалил функциональность расширенной аналитики. SQL запросы были написаны с учетом таблиц, которых нет в БД RM (categories, created_at).
- **Решение:** Восстановлена полная структура расширенной аналитики: подвкладки (Даш-борд, Тепловая карта, Статистика по оборудованию, Статистика по пользователям, Временные паттерны, Прогноз загрузки, Рекомендации), интерактивные подсказки, настраиваемый дашборд. Исправлены SQL запросы: убраны JOIN с таблицей categories, поле created_at заменено на b.date::timestamp. Исправлена логика получения данных: демо-данные используются только при ошибке подключения, а не при отсутствии данных. Добавлены информационные заметки о периоде данных (6 месяцев) на всех подвкладках. Исправлена логика доступа: для неавторизованных пользователей скрыты "Управление бронированием" и "Календарь бронирований".
- **Статус:** выполнено.

## 2025-11-26 — Автоматическая остановка предыдущих экземпляров сервера
- **Проблема:** При запуске нового экземпляра сервера предыдущие экземпляры оставались активными, что приводило к конфликтам портов и нестабильной работе.
- **Подпроект:** WSB_portal
- **Влияние:** Стабильность работы сервера, предотвращение конфликтов портов
- **Причина:** Необходимость автоматически очищать порт 8090 перед запуском нового экземпляра сервера.
- **Решение:** Реализована функция `stop_previous_instances()` в `app/main.py`, которая при запуске приложения автоматически находит и останавливает все процессы Python, слушающие порт 8090. Функция работает на Windows (netstat + taskkill) и Linux/Mac (lsof + kill). Вызывается в событии `startup_event()` перед запуском планировщика.
- **Статус:** выполнено.

## 2025-11-26 — Реализация календаря бронирований
- **Проблема:** Отсутствовал удобный способ просмотра бронирований на месяц для планирования и анализа занятости оборудования.
- **Подпроект:** WSB_portal
- **Влияние:** Пользователи портала, планирование работы
- **Причина:** Необходимость визуального представления бронирований на месяц для удобного планирования.
- **Решение:** Создана новая вкладка "Календарь бронирований" с месячным календарем. Реализована функция `get_calendar_overview()` в `bookings.py` для получения статистики бронирований по дням месяца. Добавлен API endpoint `/api/bookings/calendar`. Календарь отображает количество бронирований на каждый день, визуально выделяет дни с бронированиями. При клике на дату автоматически открывается вкладка "Управление бронированием" с выбранной датой и загружаются данные. Календарь доступен как авторизованным, так и неавторизованным пользователям (для неавторизованных показываются все бронирования).
- **Статус:** выполнено.

## 2025-11-20 — Система авторизации пользователей
