# Soft_IPK — многоуровневый репозиторий

Soft_IPK объединяет десятки подпроектов (боты, веб-приложения, AI-сервисы, парсеры). Все работы ведутся из корневой директории `C:\Soft_IPK`, чтобы переиспользовать модули между подпроектами.

## Структура верхнего уровня
- `WSB/` — Telegram-бот бронирования рабочих мест (основной код в `wsb_bot/`)
- `telegram_bot_WorkSpaceBooking/` и `NewBot/` — альтернативные реализации WSB
- `WSB_portal/` — новый веб-портал бронирования (HTTP 192.168.1.139:8090)
- `ChipDip/`, `Chip_and_Dip/`, `CRB/`, `T-Helper/`, `RAGFlow/` — другие крупные направления
- `PROJECT_CONTEXT.md`, `SOLUTIONS_HISTORY.md`, `CHAT_HISTORY.md`, `SYNC_INSTRUCTIONS.md`, `CHANGELOG.md` — корневые файлы контекста

## Правила работы
1. Всегда читать и обновлять файлы контекста на текущем уровне и в подпроекте.
2. Проверять существующий код перед добавлением новых модулей, чтобы не дублировать решения.
3. Перед изменениями описывать план, после — фиксировать результаты и линтеры.
4. Использовать Conventional Commits и документировать изменения в `CHANGELOG.md`.
5. При работе с подпроектами создавать/обновлять аналогичный набор контекстных файлов.
