## Синхронизация
- Перед работой: `git pull` из актуальной ветки (develop/feature-ветка).
- Рабочая ветка: создавать `feature/<название>` для новых задач.
- После изменений: проверить `git status`, `git diff`, добавить осмысленные коммиты по Conventional Commits.
- Обновлять контекстные файлы при изменениях логики или договоренностей.
- Перед PR: убедиться в корректности `.env` (не коммитить секреты), прогнать линтеры/тесты.
