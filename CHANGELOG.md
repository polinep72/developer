# Changelog

Все значимые изменения в проекте Soft_IPK будут документироваться в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
и этот проект придерживается [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- Создана система управления контекстом для многоуровневого проекта
- Добавлены файлы контекста для корневого уровня:
  - PROJECT_CONTEXT.md - описание архитектуры проекта
  - SOLUTIONS_HISTORY.md - история решений
  - CHAT_HISTORY.md - история переписки с AI
  - SYNC_INSTRUCTIONS.md - инструкции по синхронизации
  - .cursorrules - правила проекта
  - CHANGELOG.md - история изменений
- Документирована структура всех подпроектов
- Создана система синхронизации между компьютерами

### Changed
- Унифицированы подходы к управлению контекстом проекта
- Стандартизированы файлы контекста для всех уровней проекта

### Fixed
- Решена проблема отсутствия структурированного контекста для большого проекта
- Устранена необходимость повторного объяснения контекста при работе на разных компьютерах

## [0.1.0] - 2025-01-27

### Added
- Инициализация проекта Soft_IPK
- Создание базовой структуры многоуровневого проекта
- Анализ и документирование всех подпроектов:
  - Парсинг и автоматизация (Chip_and_Dip, ChipDip, СИ)
  - Telegram боты (CRB, WSB, NewBot, nl)
  - Веб-приложения (Cristal, dash, Zotero)
  - AI системы (T-Helper, RAGFlow)
  - Системы управления (WorkControl, backend/frontend)

### Changed
- Унифицированы подходы к разработке во всех подпроектах
- Стандартизированы файлы контекста

### Fixed
- Решена проблема фрагментированности контекста между подпроектами
- Устранена необходимость повторного анализа структуры проекта

## Типы изменений

- **Added** - для новых функций
- **Changed** - для изменений в существующей функциональности
- **Deprecated** - для функций, которые скоро будут удалены
- **Removed** - для удаленных функций
- **Fixed** - для исправлений ошибок
- **Security** - для исправлений уязвимостей

## Формат записей

### Структура записи:
```
## [Версия] - YYYY-MM-DD

### Added
- Описание добавленной функциональности

### Changed
- Описание изменений в существующей функциональности

### Fixed
- Описание исправленных ошибок
```

### Примеры записей:
```
### Added
- Новая функция парсинга для ChipDip проекта
- Интеграция с Telegram Bot API для WSB бота

### Changed
- Обновлена архитектура RAGFlow системы
- Улучшена производительность T-Helper AI помощника

### Fixed
- Исправлена ошибка подключения к базе данных в CRB боте
- Устранена проблема с загрузкой файлов в Zotero импортере
```
